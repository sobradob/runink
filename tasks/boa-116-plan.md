# BOA-116 — Revamp homepage: clarity, maps, themes, pricing, social proof

Ticket: https://linear.app/boaz-personal/issue/BOA-116
Branch: sobradob/boa-116-runink-revamp-homepage-clarity-pricing-transparency-and

## Goal / acceptance criteria

Turn the single-screen onboarding (App.tsx:245–352) into a **mobile-first, scrollable
landing page** that reduces purchase uncertainty. Done when an unauthenticated visitor on
a phone can, without connecting Strava:

1. **Understand what they're buying** within the first screen — see an actual map poster.
2. **See example route maps prominently** (the asset the feedback singled out).
3. **See the available themes** in a dedicated showcase.
4. **See pricing** ($25–$75 tiers) before connecting Strava.
5. **See social proof** — two placeholder testimonials (Alex Kidwell, Tim Groot).
6. The Strava connect CTA remains the primary action; auth-recovery alerts still work.

## Decisions (confirmed 2026-06-17)

- **Scope:** full redesign, **mobile-first** (design for ~380px, enhance up).
- **Map assets:** static export PNGs (extend `scripts/gen-mode-examples.mjs`).
- **Social proof:** real quotes, placeholders for now:
  - Alex Kidwell — "This is a cool idea"
  - Tim Groot — "This is enough proof… for me to connect my Strava"

## What exists (grounding)

- Landing = unauthenticated branch in `src/App.tsx:245–352` (no separate route/page).
- Pricing tiers: `src/features/checkout/ui/tiers.ts` ($25/$35/$55/$45/$75). NOT shown pre-auth.
- Themes: `src/data/themes.json` (10) + `themeRepository.ts`; live preview via
  `src/features/theme/ui/RoutePreview.tsx`.
- Example assets: `public/assets/examples/{single,composite}.png`, generated by
  `scripts/gen-mode-examples.mjs` from demo data — reuse/extend this.
- Stack: Vite + React 19 + React Router + Tailwind v4. Fonts: Bebas Neue / Space Grotesk.
- No social-proof component exists yet.

## Approach

Keep the redesign self-contained: extract the landing markup out of `App.tsx` into a new
`src/features/onboarding/ui/LandingPage.tsx`, composed of small section components. App.tsx
just renders `<LandingPage ...>` in the unauthenticated branch, passing the existing
connect handlers + authIssue so recovery alerts and analytics are preserved. Lower blast
radius than editing the 100-line JSX block inline, and easier to verify per-section.

## Results (2026-06-17)

Shipped the mobile-first landing page. Implemented:
- New `src/features/onboarding/ui/landing/LandingPage.tsx` (+ `landingData.ts` for the
  two placeholder testimonials). Sticky header → hero (single-run poster + connect CTA +
  auth-recovery alerts) → "two ways" examples → 10-theme showcase strip → pricing (read
  live from `tiers.ts`, $25–$75) → social proof → final CTA → footer.
- `src/App.tsx`: replaced the ~108-line inline onboarding block with `<LandingPage>`,
  threading connect handlers, `authIssue`, `error`, `logoLongPress`, gift/privacy nav.
- `scripts/gen-landing-themes.mjs`: renders the 102 km demo run across all 10 themes →
  `public/assets/landing/themes/{id}.png` (10 files, ~1.9 MB total, lazy-loaded).
- `scripts/gen-landing-examples.mjs`: the prominent hero + example-card images, each in a
  *different* colourful theme so the page isn't black-and-white only (per follow-up
  feedback) → `public/assets/landing/{hero,example-single,example-composite}.png`:
  - hero = London composite, Coral
  - single card = 102 km run, Japanese Ink
  - composite card = Budapest composite, Midnight Blue (a different city from the hero)
  (The old Noir `assets/examples/{single,composite}.png` stay in use by ModeSelect.)

Verification:
- `npm run build` (tsc -b + vite) exits 0. `npx tsc -b` clean.
- eslint on new files clean; App.tsx shows only the 2 pre-existing baseline warnings
  (lines 66, 178) — none introduced.
- Rendered the production `dist/` with no backend (forces disconnected state) and
  screenshotted at 375×812 and 1280×800: all sections present, pricing matches tiers
  ($25/$35/$55/$45/$75), both testimonials shown, theme thumbnails correctly recolored
  (verified neon = cyan-on-dark, japanese-ink = red-on-paper). No page errors beyond the
  intentional `/auth` 404 used to simulate the logged-out state.

Note: the preview MCP server couldn't start in this environment (`EPERM: uv_cwd` in the
npm runner) — verified via Playwright against the built bundle instead.
